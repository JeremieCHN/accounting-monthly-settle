# 月度汇算小工具 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 背景
餐饮企业每月需要进行物料出入库的汇算清查，根据先入先出法（FIFO）计算期末库存和出库成本。目前依赖手工计算，效率低且易出错。

### 1.2 目标
开发一个汇算小工具，用户选择 Excel 文件中的 sheet 作为输入，工具自动按先入先出法计算，并将结果输出到新文件。

### 1.3 核心用户
财务/仓管人员

---

## 2. 数据模型

### 2.1 输入数据（3个来源）

| 来源 | 必需列 | 说明 |
|------|--------|------|
| 入库记录 | 进货日期、物料名称、数量、单价、税率、金额 | 按日期排序的入库流水，数量可能为负（退货） |
| 期初库存 | 物料名称、数量、单价 | 本月期初结存 |
| 出库记录 | 物料名称、数量 | 本月出库汇总 |

### 2.2 输出数据（2个结果）

| 输出 | 列 | 说明 |
|------|----|------|
| 期末库存 | 物料名称、数量、单价 | FIFO消耗后剩余库存，单价为加权平均单价 |
| 出库成本 | 物料名称、单价 | FIFO消耗的加权平均出库单价 |

## 3. 功能设计

### 第一步：文件选择与数据预览

#### 3.1 选择输入文件

3个输入来源可能来自不同的 xlsx 文件，需分别选择：

| 输入来源 | 用户操作 | 说明 |
|----------|----------|------|
| 入库记录 | 选择一个 xlsx 文件 | 必选 |
| 期初库存 | 选择一个 xlsx 文件 | 必选 |
| 出库记录 | 选择一个 xlsx 文件 | 必选 |

- 每个输入来源有独立的"选择文件"按钮
- 不同输入来源可以选择同一个文件，也可以选择不同文件
- 选择文件后，读取该文件所有 sheet 名称，展示在下拉列表中供用户选择

#### 3.2 Sheet 映射

每个文件选中后，用户需为该文件指定对应的 sheet：

| 输入来源 | 用户操作 | 说明 |
|----------|----------|------|
| 入库记录 | 选择一个 sheet | 必选 |
| 期初库存 | 选择一个 sheet | 必选 |
| 出库记录 | 选择一个 sheet | 必选 |

- 选择 sheet 后立即预览该 sheet 的前 20 行数据

#### 3.3 列映射

每个 sheet 选中后，工具自动尝试匹配列名。匹配规则：
- 精确匹配示例中的列名（如"物料名称"、"数量"、"单价"等）
- 匹配失败时，用户手动从下拉列表中选择对应列

| 输入来源 | 必须映射的列 |
|----------|-------------|
| 入库记录 | 进货日期、物料名称、数量、单价 |
| 期初库存 | 物料名称、数量、单价 |
| 出库记录 | 物料名称、数量 |

- 未映射的列自动忽略
- 税率、金额列为非必需列，不影响 FIFO 计算

#### 3.4 数据预览

- 展示映射后的数据预览表（仅显示已映射的列）
- 默认显示前 20 行，底部提供"加载更多"按钮，点击后展示该 sheet 全部数据
- 显示总行数、物料种类数
- 标注异常数据：数量为负、日期为空、单价为0等

---

### 第二步：FIFO 计算与输出

#### 3.5 先入先出法计算逻辑

**核心算法：**

对每种物料，维护一个入库批次队列（先进先出）：

1. **初始化**：将期初库存作为第一个批次入队 `(数量, 单价)`
2. **入库处理**：按进货日期升序，将入库记录逐条入队
   - 数量为正：新增批次 `(数量, 单价)`
   - 数量为负（退货）：从队尾开始扣减（即退最近一批的货）
3. **出库处理**：从队首开始消耗
   - 若出库数量 ≤ 队首批次数量：队首数量减少，出库成本 = 出库数量 × 队首单价
   - 若出库数量 > 队首批次数量：消耗队首整批，继续消耗下一批，直到满足出库数量
   - 出库成本 = 各批次消耗数量 × 对应单价的加权和
4. **计算结果**：
   - 期末库存 = 队列中剩余批次汇总（总数量, 加权平均单价）
   - 出库单价 = 出库总成本 / 出库总数量（加权平均）

**边界情况处理：**

| 场景 | 处理方式 |
|------|----------|
| 出库数量 > 可用库存 | 标记警告，按实际库存计算，不足部分单价记为0 |
| 入库记录中数量为负 | 视为退货，从最近批次扣减 |
| 某物料仅有期初无入库/出库 | 期末库存 = 期初库存 |
| 某物料有入库无出库 | 期末库存 = 期初 + 入库合计 |
| 进货日期为空 | 排在最后处理 |
| 同一物料同日多笔入库 | 按在表格中的行顺序入队 |

#### 3.6 结果展示

- 在界面上展示计算结果预览
- 期末库存表：物料名称、数量、加权平均单价、金额（=数量×单价）
- 出库成本表：物料名称、出库数量、加权平均单价、出库总金额
- 标注有异常的物料（如出库超过库存的）

#### 3.7 输出到文件

- 点击"导出结果"按钮，将结果写入新的 xlsx 文件
- 输出文件包含2个 sheet：
  - `期末库存`：物料名称、数量、单价
  - `出库成本`：物料名称、单价
- 用户选择输出文件路径和文件名
- 默认文件名：`汇算结果_YYYYMMDD.xlsx`

## 4. 技术方案

### 4.1 技术栈

- **GUI**：tkinter（Python 内置，无需额外安装）
- **Excel 读写**：openpyxl
- **打包部署**：可选 PyInstaller 打包为 exe，或直接 `python main.py` 运行

### 4.2 项目结构

```
月度汇算/
├── src/
│   ├── main.py             # 主程序入口 & GUI 界面
│   ├── excel_handler.py    # Excel 读写逻辑
│   └── fifo_engine.py      # FIFO 计算引擎
├── run.bat                 # 开发运行脚本
├── build.bat               # PyInstaller 打包脚本
├── requirements.txt        # 依赖
└── 汇算示例表.xlsx         # 示例数据
```

### 4.3 GUI 布局

使用 tkinter 构建单窗口应用，分步骤切换页面：

- **页面1 - 文件选择**：3组"选择文件"按钮 + sheet 下拉框，分别对应入库/期初/出库 → 选择后预览数据
- **页面2 - 列映射与预览**：自动匹配列名 + 手动下拉调整 → 数据预览表格（默认20行 + "加载更多"按钮）
- **页面3 - 计算结果**：展示期末库存和出库成本表格 → "导出结果"按钮选择保存路径
